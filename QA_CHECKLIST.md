# Hanspoon Frontend 기능 점검 체크리스트

목표: 기능이 깨지지 않았는지 빠르게 확인한다.

## 1. 사전 준비

1. 백엔드 서버 실행
2. 프론트 실행: `npm run dev`
3. 점검 계정 준비
- 일반 사용자 1개
- 관리자 사용자 1개

## 2. 공통 기본 점검

1. 홈(`/`) 접속 시 화면이 정상 렌더링된다.
2. 상단 메뉴/링크 클릭 시 콘솔 에러 없이 라우팅된다.
3. 새로고침 후에도 치명 에러 없이 화면이 유지된다.

## 3. 인증

1. 로그인(`/login`)
- 정상 계정으로 로그인 성공
- 로그인 후 홈 이동 확인
2. 로그아웃
- 로그아웃 후 사용자 메뉴가 비로그인 상태로 변경
3. 보호 라우트
- 비관리자가 `/admin/*` 직접 접근 시 차단

## 4. 상품/장바구니/주문

1. 상품 목록(`/products`)
- 카테고리/정렬/검색 동작
2. 상품 상세(`/products/:id`)
- 수량 조절, 장바구니 추가
3. 장바구니(`/cart`)
- 수량 변경, 개별 삭제, 전체 비우기
4. 주문 생성
- 주문 생성 후 주문 상세(`/orders/:orderId`) 이동
5. 주문 상세
- 결제 이동 버튼 동작
- 상태 액션 버튼(SHIP/DELIVER/CANCEL) 동작

## 5. 결제

1. 결제 페이지(`/payment`)
- 주문/예약에서 전달된 금액/상품명이 표시됨
2. 결제 성공/실패 페이지 이동
- 성공: `/payment/success`
- 실패: `/payment/fail`

## 6. 마이페이지

1. 프로필(`/mypage/profile`) 조회/수정
2. 주문(`/mypage/orders`) 목록과 상세 이동
3. 찜(`/mypage/wishlist`) 목록/해제
4. 리뷰(`/mypage/reviews`) 목록/수정/삭제
5. 문의(`/mypage/inquiries`) 목록/수정/삭제
6. 클래스 예약(`/mypage/reservations`) 조회/취소요청

## 7. 원데이 클래스

1. 목록(`/classes/oneday/classes`) 조회/필터
2. 상세(`/classes/oneday/classes/:classId`)
- 찜 토글
- 홀딩 예약
- 바로 결제 이동
- 리뷰 작성/삭제(권한 조건 포함)
3. 내 예약(`/classes/oneday/reservations`)
- 필터/페이징
- 상세 조회
- HOLD 상태 결제 이동
- 취소 처리

## 8. 레시피

1. 등록(`/recipes`)
- 필수값 입력, 저장 성공
2. 목록(`/recipes/list`)
- 카드 렌더링, 상세 이동
3. 상세(`/recipes/:id`)
- 렌더링, 수정 버튼 이동, 삭제 동작
4. 수정(`/recipes/edit/:id`)
- 기존 데이터 로드, 저장 성공
5. 삭제 목록(`/recipes/deletelist`)
- 복원 동작

## 9. 공지/FAQ

1. 공지 목록(`/notice`) 조회
2. 공지 상세(`/notice/:id`) 조회
3. FAQ(`/faq`) 조회

## 10. 관리자

1. 공지 관리(`/admin/notice`)
- 목록/작성/수정/삭제
2. FAQ 관리(`/admin/faq`)
- 목록/작성/수정/삭제
3. 사용자 관리(`/admin/users`)
- 목록 조회, 상세 이동
4. 결제 관리(`/admin/payments`)
- 목록 조회, 환불 처리

## 11. 회귀 확인 기준

1. 브라우저 콘솔에 에러가 없어야 한다.
2. 네트워크 요청 4xx/5xx는 의도된 경우 외 발생하지 않아야 한다.
3. 동일 동작을 2회 반복해도 상태 꼬임이 없어야 한다.

## 12. 최근 정적 검사 결과

1. `npm run lint` 통과
2. `npm run build` 통과
